# iGaming Fintech Foundry Example

This file is a non-authoritative convenience note. Foundry does not load it as part
of the governed spec-kit context.

Use these project entry points instead:

- `AGENTS.md` for project orientation and retrieval posture
- `docs/adrs/` for accepted domain and architecture decisions
- `docs/regulations/` for compliance requirements and test tags
- `docs/runbooks/` for operational procedures and failure handling
- `mix foundry.project.context` for live code-derived system topology
- `mix foundry.project.status` for current health, stack, sensitive resources, and
  compliance state

Do not add durable requirements, decisions, or operational procedures here. Put them
in the appropriate spec-kit artifact so Foundry can index and reason over them.
